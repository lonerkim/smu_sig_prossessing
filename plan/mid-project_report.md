### 2.2 최적 파이프라인 조합

**현재까지 확인된 최적 순서**:
```
1. Median Filter (k=3)        → Impulse 잡음 제거
2. Wiener Filter (var=400)   → Gaussian 잡음 + residual noise 제거
3. FFT Notch (99.5%)         → 주기적 잡음(스캔라인, 모터) 제거
4. Channel Correction        → RGB 불균형 보정
5. Gamma Correction (γ=1.8)  → 전체 밝기/대비 개선
6. Histogram Eq (YUV)        → 최종 대비 최적화
```

**이유**:
1. **Median → Wiener**: Impulse 잡음 먼저 제거 후 Gaussian 잡음 처리
2. **FFT Notch**: 주파수 도메인에서 주기적 잡음 제거 (공간 도메인 필터로는 제거 어려움)
3. **Channel → Gamma**: 색상 보정 후 밝기 조절 (순서 중요)
4. **Histogram Eq 마지막**: 최종 단계에서 대비 최적화

### 3.2 알고리즘 deepest Dive

#### 1. Wiener Filter 구현

**수학적 모델**:
```
H(u,v) = S(u,v) / (S(u,v) + N(u,v))
```
- S(u,v): 신호 파워 스펙트럼
- N(u,v): 잡음 파워 스펙트럼

**구현 특징**:
- 주파수 도메인에서 처리 (FFT 기반)
- 신호 파워 스펙트럼을 평균값으로 추정
- 잡음 분산을 파라미터로 입력 (default: 625 = 25²)

#### 2. FFT Notch Filter 구현

**처리 과정**:
1. 2D FFT 변환 → 주파수 도메인으로 변환
2. 스펙트럼의 percentile threshold (99.5%)로 노이즈 피크 탐지
3. DC 성분 (0,0) 보호 (중요!)
4. 노이즈 피크 위치의 값을 0으로 설정
5. IFFT → 공간 도메인으로 복원

**특징**:
- 주기적 잡음(스캔라인, 모터 잡음)에 특화
- 공간 도메인 필터로는 제거가 어려운 고주파 잡음 제거 가능

#### 3. Channel Correction 구현

**알고리즘**:
1. 각 RGB 채널의 평균 값을 계산
2. 모든 채널의 평균 평균값 (target) 계산
3. 각 채널의 scale factor = target / channel_mean
4. Scale factor를 0.7-1.3 범위로 clamping (과도한 보정 방지)
5. 각 채널에 scale factor 적용

**장점**:
- 실제 색을 정확히 복원하는 것보다 시인성 개선에 초점
- 과도한 보정으로 인한 부자연스러운 결과 방지

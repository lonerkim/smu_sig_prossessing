## 1. 프로젝트 목표 (한 줄)

아날로그 FPV DVR 영상에서 시인성을 해치는 **잡음·색왜곡·대비저하**를 수업 기법으로 완화하는 범용 파이프라인을 설계·구현·평가한다.

**범위 (Scope)**
- ✅ 할 것: Gaussian/Impulse 잡음 제거, 색상 채널 보정, 감마/로그/히스토그램 평활화, FFT 기반 주기적 잡음 제거, 기본 보간
- ❌ 안 할 것: 심층학습 디노이징, 완전히 손실된 프레임 복원, 실시간 처리

## 2. 잡음 패턴 분류 (빈도순)

> 실제 FPV DVR footage에서 마주치는 빈도 기준. **Tier 1**은 거의 항상 보이고, **Tier 4**는 극단적 환경에서만 발생.

### Tier 1 — 항상 존재 (Every Frame)

| 패턴 | 원인 | 모델 | 처리 기법 |
|---|---|---|---|
| **루미넌스 잡음** (Luminance / AWGN) | 센서 열잡음, 증폭기 | N(0, σ²) 가우시안 | 위너 필터, 로우패스 필터 |
| **크로미넌스 잡음** (Chrominance) | 센서+컴포지트 분리 불량 | 채널별 독립 가우시안 | 채널별 로우패스, 공격적 디노이징 |
| **대비·밝기 저하** | 조도 변화, VTX 게인 | 감마 왜곡, 히스토그램 축소 | 감마 보정, 히스토그램 평활화, 로그 연산 |
| **색상 채널 불균형** | NTSC 디코더 편차 | RGB 채널별 offset/gain | 채널별 밝기 분포 보정 |

### Tier 2 — 자주 발생 (Typical Flight)

| 패턴 | 원인 | 모델 | 처리 기법 |
|---|---|---|---|
| **Impulse / Salt & Pepper** | 센서 결함, ADC 오류 | 확률 p로 픽셀 min/max | 미디언 필터 |
| **도트 크롤** (Dot Crawl) | 3.58MHz 크로마 누설 → 루마 | 주기적 수평 점선 | 2D FFT 노이즈 피크 제거, 콤 필터 |
| **크로스 컬러 / 레인보우** | 고주파 루마 → 크로마 대역 침범 | 무지개 일렁임 | 주파수 분리, 크로마 로우패스 |
| **색번짐 / 지연 오차** (Chroma Delay) | 루마-크로마 시간지연 차이 | 색 윤곽선 어긋남 | 채널 시프트 보정 |

### Tier 3 — 간헐적 (Poor Conditions)

| 패턴 | 원인 | 모델 | 처리 기법 |
|---|---|---|---|
| **고스팅** (Ghosting) | 다중경로 반사 신호 중복 | offset 덧셈 | 위너 디컨볼루션, 적응 필터 |
| **사선 밴딩** (Diagonal Lines / Motor Noise) | ESC 스위칭, 2.4GHz 간섭 | 주기적 대각선 파형 | 2D FFT notch 필터 |
| **링잉** (Ringing / Gibbs) | 급격한 엣지 + 저가 필터 | 엣지 에코 진동 | 로우패스, 엣지 보존 필터 |
| **스캔라인 불균일** | VTX 수평 동기 불안정 | 수평줄 밝기 편차 | FFT + 행단위 정규화 |

### Tier 4 — 드묾 (Extreme / Edge-of-Range)

| 패턴 | 원인 | 모델 | 처리 기법 |
|---|---|---|---|
| **스노우 / 스태틱** (Dropout) | 신호 완전 손실, SNR 붕괴 | 전면 흑백 난수 | 프레임 스킵, 인접프레임 보간 |
| **티어링** (Tearing) | 동기 펄스 손상 | 수평 찢어짐 | 수평 동기 복원, 보간 |
| **롤링** (Rolling) | 수직 동기 상실 | 화면 세로 흘러내림 | 수직 동기 복원 |

## 3. 구현 파이프라인

```
┌─────────────┐    ┌──────────────┐    ┌───────────────┐    ┌──────────────┐    ┌──────────┐
│  입력 영상   │───→│ 잡음 분류    │───→│ 잡음 제거     │───→│ 색상/대비    │───→│  평가    │
│ (Degraded)  │    │ (Frame 상태) │    │ (Filter 선택) │    │ 보정         │    │ PSNR etc │
└─────────────┘    └──────────────┘    └───────────────┘    └──────────────┘    └──────────┘
```

### Phase 0: 실험 데이터 준비
- [ ] 깨끗한 디지털 이미지 5–10장 선정 (다양한 밝기/색상)
- [ ] 인위 열화 함수 구현
  - [ ] Gaussian noise 추가 (σ 제어)
  - [ ] Impulse noise 추가 (확률 p 제어)
  - [ ] 밝기/대비 저하 (감마·곱셈)
  - [ ] 색상 채널 편향 (채널별 gain/offset)
  - [ ] 주기적 잡음 (사인파 오버레이 — Tier 2/3 시뮬)
- [ ] 실제 FPV DVR footage에서 대표 프레임 10–20장 추출

### Phase 1: 잡음 제거 (Tier 1–2 우선)

**1-A. Impulse 잡음 → 미디언 필터**
- [ ] `cv2.medianBlur()` 커널 크기 3, 5 비교
- [ ] Salt & pepper 시나리오에서 PSNR/SSIM 측정

**1-B. Gaussian 잡음 → 로우패스 + 위너 필터**
- [ ] 가우시안 로우패스: `cv2.GaussianBlur()` σ 변화
- [ ] 위너 필터 구현 (잡음 분산 추정 → 역필터링)
- [ ] 두 방법 PSNR/SSIM 비교, 엣지 보존 여부 확인

**1-C. 주기적 잡음 → 2D FFT**
- [ ] FFT 스펙트럼에서 노이즈 피크 탐지
- [ ] Notch 필터로 피크 제거 → IFFT
- [ ] 도트 크롤·사선 밴딩 시나리오에 적용

### Phase 2: 색상 및 대비 보정 (Tier 1)

**2-A. 감마 보정**
- [ ] γ < 1 (어두운 영상 밝게) vs γ > 1 비교
- [ ] 시각적 결과 + 히스토그램 변화 기록

**2-B. 로그 연산**
- [ ] 어두운 영역 대비 강조 효과 확인
- [ ] 감마 보정과 결과 비교

**2-C. 히스토그램 평활화**
- [ ] 그레이스케일: `cv2.equalizeHist()`
- [ ] 컬러: YUV 공간에서 Y채널만 평활화 → RGB 복원
- [ ] 과도한 평활화(자연스럽지 않음) 한계점 기록

**2-D. 채널별 보정**
- [ ] RGB 각 채널 히스토그램 분석
- [ ] 편향 채널 gain/offset 조정
- [ ] 크로미넌스 잡음은 공격적 로우패스 적용

### Phase 3: 보간 (Tier 3–4 보조)

- [ ] 손상 영역 마스킹 → bilinear 보간
- [ ] bicubic 보간과 비교
- [ ] 손실 심한 프레임 → 인접 프레임 temporal 보간 (선택)

### Phase 4: 실제 Footage 적용

- [ ] 프레임 상태 진단 (잡음 강도→필터 선택 로직)
- [ ] Phase 1–2 파이프라인 실제 DVR 프레임에 적용
- [ ] 전/후 시각 비교 + 히스토그램 비교 정리

## 4. 평가 계획

| 대상 | 방법 | 지표 |
|---|---|---|
| 인위 열화 영상 | 정량 평가 | PSNR, SSIM (원본 대비) |
| 실제 DVR 영상 | 정성 평가 | 전/후 이미지 나란히 비교 |
| 전체 | 보조 | 히스토그램 전/후 비교 |

**비교 매트릭스:**
- 잡음 제거: 미디언 vs 로우패스 vs 위너 (각 시나리오별)
- 대비 보정: 감마 vs 로그 vs 히스토그램 평활화
- 조합: 최적 조합 파이프라인 vs 개별 적용
